<Section icon={Wrench} title="Technical Requirements & Mod Installation">

Before diving into the Nexus, it is crucial to understand how Urban Strife handles external files. Since the game utilizes the Unreal Engine, most mods are distributed as `.pak` files. These files contain the modified assets or logic that override the base game data. Following a standardized installation process ensures that your game remains stable even when running multiple active mods.

<Steps>
  <Step num="1" title="Locate Game Directory">
    Navigate to your Urban Strife installation folder. This is typically found under `SteamLibrary/steamapps/common/Urban Strife`.
  </Step>
  <Step num="2" title="Identify the Paks Folder">
    Open the `UrbanStrife` folder, then go to `Content` and finally `Paks`. This is where the core game data resides.
  </Step>
  <Step num="3" title="Create a ~mods Subfolder">
    Inside the `Paks` directory, create a new folder named `~mods` (the tilde is essential). This tells the Unreal Engine to load these files with higher priority.
  </Step>
  <Step num="4" title="Deploy Mod Files">
    Download your chosen mod from Nexus. Extract the `.pak` file and move it directly into the `~mods` folder you just created.
  </Step>
  <Step num="5" title="Verify Installation">
    Launch the game. Check the version number or specific modded features (like increased carry weight) to ensure the mod is active.
  </Step>
</Steps>

<Callout type="warning" title="Version Mismatch">
Always check if a mod is compatible with version **1.03 or higher**. Older mods designed for early access builds may cause the game to crash during the Little Italy or Old Port transitions.
</Callout>

| Mod Tool | Function | 2026 Compatibility |
| :--- | :--- | :--- |
| **UE4SS** | Scripting framework for advanced mods | Fully Supported |
| **Nexus Vortex** | Automated installation and load order | Recommended |

<Section icon={Shield} title="Stability and Conflict Management">

Running multiple mods in Urban Strife requires a careful approach to file management. Because most mods modify the same core tables (like `GlobalSettings.uasset`), conflicts are inevitable. If two mods attempt to change the same weapon's damage, the one loaded last in the `~mods` folder (alphabetically) will usually take precedence.

**Best Practices for a Stable Game:**
1. **Load Order**: Rename your favorite mods with a prefix like `z_` to ensure they load last and override others.
2. **One at a Time**: Install mods one by one and test the game after each installation to identify which one causes issues.
3. **Check the Logs**: If the game crashes, check the `UrbanStrife/Saved/Logs` folder for clues on which asset failed to load.

<Callout type="warning" title="Save File Integrity">
Mods that alter the inventory structure or add new item IDs are high-risk. If you uninstall such a mod, your save file may become corrupted if your characters are currently holding modded items.
</Callout>

| Issue | Potential Cause | Solution |
| :--- | :--- | :--- |
| **Infinite Loading** | Conflict in map scripts | Remove recent map mods |
| **Invisible Items** | Missing texture assets | Reinstall the mod base files |
| **Crash on Combat** | AP or AI logic error | Check UE4SS script console |

</Section>
