<Section icon={Settings} title="How to Update the Table Manually">

Since Urban Strife is in active development, game updates often break Cheat Engine tables by changing memory offsets. The **urban strife cheat engine** can be manually updated if you understand the basic logic of assembly code. The video tutorial by Big Bear highlights that most Unreal Engine updates only change the "base pointer" rather than the entire logic of the script.

### Troubleshooting and Updates

| Issue | Cause | Solution |
| :--- | :--- | :--- |
| **Script won't activate** | AOB (Array of Bytes) changed | Perform a new AOB scan for the function |
| **Values show as ??** | Game version mismatch | Update the base pointer in the script |
| **Game Crashes** | Memory address corruption | Disable "Freeze" on health or AP values |

<Steps>
  <Step num="1" title="Identify the Script">
    Right-click the failing script (e.g., Base Control) and select "Change Script."
  </Step>
  <Step num="2" title="Find the Injection Point">
    Look for the line moving a register (like `rcx` or `rdx`) into a base pointer.
  </Step>
  <Step num="3" title="Compare Registers">
    If the game updated, the register might have changed from `rcx` to `rax`. Update the script to match the new register used by the game's code.
  </Step>
  <Step num="4" title="Re-save and Test">
    Save the script and attempt to toggle it. If it stays checked, the update was successful.
  </Step>
</Steps>

<Callout type="tip" title="Memory Viewer Tip">
Use the "Memory View" function in Cheat Engine to see the surrounding code. Often, the game developers only move the function's location, and the actual logic remains identical, making it easy to fix with a simple copy-paste of the new address.
</Callout>
