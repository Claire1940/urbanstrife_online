<Section icon={Wrench} title="Prérequis techniques et installation des mods">

Avant de vous plonger dans le Nexus, il est crucial de comprendre comment Urban Strife gère les fichiers externes. Le jeu utilisant l'Unreal Engine, la plupart des mods sont distribués sous forme de fichiers `.pak`. Ces fichiers contiennent les éléments modifiés ou la logique qui remplacent les données du jeu de base. Suivre un processus d'installation standardisé garantit la stabilité de votre jeu, même lorsque vous utilisez plusieurs mods actifs.

<Steps>
  <Step num="1" title="Localiser le répertoire du jeu">
    Accédez au dossier d'installation d'Urban Strife. Il se trouve généralement sous `SteamLibrary/steamapps/common/Urban Strife`.
  </Step>
  <Step num="2" title="Identifier le dossier Paks">
    Ouvrez le dossier `UrbanStrife`, puis allez dans `Content` et enfin `Paks`. C'est là que résident les données de base du jeu.
  </Step>
  <Step num="3" title="Créer un sous-dossier ~mods">
    Dans le répertoire `Paks`, créez un nouveau dossier nommé `~mods` (le tilde est essentiel). Cela indique à l'Unreal Engine de charger ces fichiers en priorité.
  </Step>
  <Step num="4" title="Déployer les fichiers de mod">
    Téléchargez le mod de votre choix sur Nexus. Extrayez le fichier `.pak` et déplacez-le directement dans le dossier `~mods` que vous venez de créer.
  </Step>
  <Step num="5" title="Vérifier l'installation">
    Lancez le jeu. Vérifiez le numéro de version ou des fonctionnalités modifiées spécifiques (comme l'augmentation du poids de transport) pour vous assurer que le mod est actif.
  </Step>
</Steps>

<Callout type="warning" title="Incompatibilité de version">
Vérifiez toujours si un mod est compatible avec la version **1.03 ou supérieure**. Les mods plus anciens conçus pour les versions en accès anticipé peuvent provoquer un plantage du jeu lors des transitions vers Little Italy ou Old Port.
</Callout>

| Outil de modding | Fonction | Compatibilité 2026 |

| **UE4SS** | Framework de script pour mods avancés | Entièrement pris en charge |
| **Nexus Vortex** | Installation automatisée et ordre de chargement | Recommandé |
| **Pak Unpacker** | Pour les utilisateurs souhaitant créer leurs propres mods | Utilisateurs avancés uniquement |

<Section icon={Shield} title="Stabilité et gestion des conflits">

L'exécution de plusieurs mods dans Urban Strife nécessite une approche prudente de la gestion des fichiers. Comme la plupart des mods modifient les mêmes tables de base (comme `GlobalSettings.uasset`), les conflits sont inévitables. Si deux mods tentent de modifier les dégâts d'une même arme, celui chargé en dernier dans le dossier `~mods` (par ordre alphabétique) l'emportera généralement.

**Bonnes pratiques pour un jeu stable :**
1. **Ordre de chargement** : Renommez vos mods préférés avec un préfixe comme `z_` pour vous assurer qu'ils se chargent en dernier et écrasent les autres.
2. **Un par un** : Installez les mods un par un et testez le jeu après chaque installation pour identifier celui qui pose problème.
3. **Vérifier les journaux (Logs)** : Si le jeu plante, vérifiez le dossier `UrbanStrife/Saved/Logs` pour trouver des indices sur l'élément qui n'a pas pu se charger.

<Callout type="warning" title="Intégrité des fichiers de sauvegarde">
Les mods qui modifient la structure de l'inventaire ou ajoutent de nouveaux identifiants d'objets présentent un risque élevé. Si vous désinstallez un tel mod, votre fichier de sauvegarde peut être corrompu si vos personnages possèdent actuellement des objets modifiés.
</Callout>

| Problème | Cause potentielle | Solution |
| :--- | :--- | :--- |
| **Chargement infini** | Conflit dans les scripts de carte | Supprimer les mods de carte récents |
| **Objets invisibles** | Éléments de texture manquants | Réinstaller les fichiers de base du mod |
| **Plantage en combat** | Erreur de logique de PA ou d'IA | Vérifier la console de script UE4SS |

</Section>
