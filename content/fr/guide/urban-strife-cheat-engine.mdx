<Section icon={Shield} title="Gestion des ressources de base et des fournitures">

La gestion des ressources est le pilier de la survie dans Urban Strife. La table **Cheat Engine d'Urban Strife** comprend un script de Contrôle de la Base qui se connecte aux variables globales de fournitures. Une fois activé, les joueurs peuvent voir les mises à jour en temps réel de l'inventaire de leur base. C'est particulièrement utile pour les joueurs qui souhaitent se concentrer sur le combat tactique plutôt que sur la recherche constante de carburant ou de nourriture.

| Type de ressource | Description | Action recommandée |
| :--- | :--- | :--- |
| **Matériaux** | Utilisés pour la construction et les réparations | Régler sur 9999 pour des améliorations instantanées |
| **Moral** | Affecte les performances des colons | Maintenir au-dessus de 80 pour éviter les désertions |
| **Nourriture** | Consommation quotidienne requise | Bloquer à 500 pour ignorer la faim |
| **Essence** | Requise pour les véhicules et l'énergie | Geler à des valeurs élevées pour les déplacements |

<Callout type="tip" title="Gestion du moral">
Si votre moral baisse trop à cause d'événements de l'histoire, utilisez le script de Contrôle de la Base pour le réinitialiser instantanément. Cela évite les effets de statut négatifs qui peuvent paralyser votre efficacité au combat lors des raids de début de partie.
</Callout>

Le script fonctionne en lisant les valeurs de la base de données stockées dans la mémoire du jeu. Bien que l'interface utilisateur puisse afficher les taux de production et de consommation, modifier directement le nombre brut de matériaux est le moyen le plus direct d'influencer la survie de votre base. Notez que certaines valeurs sont purement visuelles dans la base de données et peuvent ne pas affecter le jeu à moins que la fonction de consommation principale ne soit contournée.

</Section>

<Section icon={Activity} title="Édition avancée des statistiques de personnage">

Le Pointeur de Personnage est la partie la plus puissante mais aussi la plus sensible de la table **Cheat Engine d'Urban Strife**. Il permet de modifier des attributs de base tels que la force, l'agilité et la précision de tir. Cependant, comme le jeu utilise un système de pointeur dynamique, les valeurs peuvent ne pas apparaître immédiatement après l'activation du script.

| Attribut | Impact au combat | Valeur max recommandée |
| :--- | :--- | :--- |
| **Force** | Capacité de transport et dégâts de mêlée | 100 |
| **Agilité** | Points d'action (PA) totaux et esquive | 100 |
| **Précision de tir** | Précision avec les armes à feu | 100 |
| **Intelligence** | Gain d'expérience et tests de compétences | 100 |
| **Santé** | PV totaux avant la mort | 999 |

<Steps>
  <Step num="1" title="Activer le script de personnage">
    Activez l'option "Character Pointer" (Pointeur de Personnage) dans la liste de Cheat Engine.
  </Step>
  <Step num="2" title="Déclencher la mise à jour">
    Déplacez votre personnage d'une case dans le monde du jeu. Cela force le jeu à mettre à jour l'adresse mémoire recherchée par le script.
  </Step>
  <Step num="3" title="Vérifier le tag du personnage">
    Vérifiez le 'Character Tag' dans la liste CE. S'il est de 1, 2, 3 ou 4, vous modifiez probablement votre équipe principale. S'il affiche d'autres valeurs, vous regardez peut-être un PNJ ou un zombie.
  </Step>
  <Step num="4" title="Appliquer les modifications">
    Modifiez les valeurs des attributs souhaités et gelez-les si nécessaire.
  </Step>
</Steps>

<Callout type="warning" title="Le bug de la valeur 55">
Si vous voyez toutes les statistiques s'afficher à "55", c'est que le pointeur cible actuellement un PNJ générique ou ne s'est pas encore connecté à votre personnage spécifique. Ne modifiez pas ces valeurs avant d'avoir déplacé votre personnage et de voir vos statistiques réelles reflétées dans la table.
</Callout>

</Section>

<Section icon={Settings} title="Comment mettre à jour la table manuellement">

Comme Urban Strife est en cours de développement actif, les mises à jour du jeu brisent souvent les tables Cheat Engine en modifiant les décalages (offsets) de mémoire. La table **Cheat Engine d'Urban Strife** peut être mise à jour manuellement si vous comprenez la logique de base du code assembleur. Le tutoriel vidéo de Big Bear souligne que la plupart des mises à jour d'Unreal Engine ne modifient que le "pointeur de base" plutôt que toute la logique du script.

### Dépannage et mises à jour

| Problème | Cause | Solution |
| :--- | :--- | :--- |
| **Le script ne s'active pas** | AOB (Array of Bytes) modifié | Effectuez un nouveau scan AOB pour la fonction |
| **Les valeurs affichent ??** | Incompatibilité de version du jeu | Mettez à jour le pointeur de base dans le script |
| **Le jeu plante** | Corruption de l'adresse mémoire | Désactivez le gel ("Freeze") sur les valeurs de santé ou de PA |

<Steps>
  <Step num="1" title="Identifier le script">
    Faites un clic droit sur le script défaillant (ex. Base Control) et sélectionnez "Change Script" (Modifier le script).
  </Step>
  <Step num="2" title="Trouver le point d'injection">
    Cherchez la ligne qui déplace un registre (comme `rcx` ou `rdx`) dans un pointeur de base.
  </Step>
  <Step num="3" title="Comparer les registres">
    Si le jeu a été mis à jour, le registre a peut-être changé de `rcx` à `rax`. Mettez à jour le script pour correspondre au nouveau registre utilisé par le code du jeu.
  </Step>
  <Step num="4" title="Sauvegarder et tester">
    Sauvegardez le script et essayez de l'activer. S'il reste coché, la mise à jour a réussi.
  </Step>
</Steps>

<Callout type="tip" title="Astuce de l'éditeur de mémoire">
Utilisez la fonction "Memory View" de Cheat Engine pour voir le code environnant. Souvent, les développeurs du jeu ne font que déplacer l'emplacement de la fonction, et la logique réelle reste identique, ce qui facilite la correction par un simple copier-coller de la nouvelle adresse.
</Callout>

</Section>
